OIT Technician Toolkit: Master Configuration Guide

UPDATED: January 18, 2026

1. EXECUTIVE SUMMARY

The OIT Technician Toolkit is an integrated administrative platform designed to bridge SCCM, Active Directory, 25Live, Sassafras KeyServer, and HappyFox. By orchestrating these systems, the toolkit provides field technicians with a unified interface for system diagnostics and reporting, ensuring high-fidelity visualizations of day-to-day problem areas. The toolkit is optimized for both desktop and mobile environments, enabling field mobility for technicians utilizing tablets during campus-wide lab checks.

2. CORE MODULE FUNCTIONALITIES

1. Lab Wake-Up (SCCM Integrated)

Leverages SCCM's internal Wake-on-LAN protocols to broadcast magic packets across subnets, facilitating remote power-on for lab maintenance or class preparation.

2. Ghost Hunter (Sassafras API)

Uses the Sassafras KeyServer API to flag workstations failing to check in within 7 days, providing technicians with a real-time focus list for hardware remediation.

3. Campus Lab Report (Integrated)

Combines Ghost Hunter data with 25Live schedules to identify maintenance windows. Results are documented weekly via the HappyFox API to provide a proactive "jump start" for technicians.

4. AD & SCCM Purge

A "Clean Start" tool that wipes computer metadata from both Active Directory OUs and SCCM Site Databases in a single administrative action.

5. Sassafras Manager (Reporting)

Syncs 25Live academic calendars with Sassafras Time Sets. This ensures that usage reports reflect only active class hours, providing accurate visualizations for non-IT staff.

6. Room Checker

Queries 25Live Web Services to provide immediate availability and class schedule gaps for all campus labs.

3. SYSTEM ARCHITECTURE & SECURITY

The toolkit serves as a "Single-Pane-of-Glass" by translating institution-wide data into actionable IT intelligence while maintaining a robust security posture.

3.1 Identity & Access Management (IAM)

To protect these powerful administrative tools from unauthorized use or abuse, the toolkit requires Active Directory (AD) authentication. Only authorized technicians with valid credentials and specifically assigned AD group permissions can gain access to the application.

SAFEGUARD PROTOCOLS: All high-impact administrative actions (such as the AD/SCCM Purge) are gated by functional failsafe prompts. Technicians must manually confirm these actions, ensuring that critical infrastructure is protected from accidental harm.
System Integration Method Purpose
SCCM PowerShell Bridge Drives Wake-on-LAN and asset decommissioning (Purge Tool).
Active Directory LDAP / Identity Bridge Authenticates technician credentials and authorizes tool access based on group membership.
25Live JSON Web Services Synchronizes academic schedules to identify maintenance gaps and drive reporting Time Sets.
Sassafras KeyServer REST API Reports asset check-in health (Ghost Hunter) and provides data for usage visualizations.
HappyFox RESTful API Injects automated health logs and weekly summaries into active technician tickets.

4. AUTOMATED REPORTING & VISUALIZATION

4.1 Sassafras Time Set Management

To support non-IT staff and institutional stakeholders, the Sassafras Manager automates the creation of Time Sets based on 25Live course data. This ensures that usage reports ignore idle hours (nights/weekends) and provide accurate data specifically for times when classes are scheduled to run.

DATA ACCURACY: Without Time Sets, reports reflect 24/7 availability. By integrating 25Live class ranges, we deliver visualizations that represent actual academic utilization. This automation allows OIT to instantly sync accurate class schedules (Monday through Friday) for every lab on campus, saving hours of manual labor.

4.2 Proactive Weekly Ticketing

The Campus Lab Report executes on a weekly basis, utilizing the HappyFox API to document health logs in the Weekly Lab Checks ticket. This allows technicians to address persistent offline workstations (identified via the 7-day Sassafras check-in window) during identified 25Live maintenance gaps.
